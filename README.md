# 🧪 QA Test Case Generator

**Микро-SaaS приложение (MVP) для автоматической генерации тест-кейсов по описанию задачи с помощью LLM (OpenAI, Mistral AI, Groq, Ollama и др.).**

Позволяет QA-инженерам настраивать структуру тест-кейса (шаблон с динамическими атрибутами) и получать готовые кейсы, строго следующие заданной форме.
**Новая возможность:** загружайте скриншоты интерфейсов, получайте их текстовое описание через мультимодальную LLM и используйте эти описания в тест-кейсах через плейсхолдеры `{{Имя страницы}}`.

---

## ✨ Возможности

- **Конструктор шаблона** – добавляйте/удаляйте любые поля тест-кейса (название, предусловия, шаги, ожидаемый результат, окружение и т.д.)
- **Генерация через LLM** – поддерживается несколько провайдеров (OpenAI, Mistral, Groq, OpenRouter, Ollama)
- **Библиотека страниц** – загрузка скриншотов, автоматическое описание страницы через мультимодальную модель (Mistral Pixtral / LLaVA) и сохранение в SQLite
- **Плейсхолдеры в описании задачи** – пишите `{{Имя страницы}}` в поле задачи, и система подставит сохранённое описание
- **Строгая валидация ответа** – динамическая Pydantic-схема гарантирует, что модель вернёт данные только с указанными атрибутами
- **Копирование в буфер** – результат отображается в структурированном виде и может быть скопирован как JSON
- **Лёгкий фронтенд** – один HTML-файл с Tailwind CSS и ванильным JS (без фреймворков)
- **Готовность к деплою** – FastAPI отдаёт и бэкенд, и статику

---

## 🖼️ Интерфейс

1. Переключайтесь между вкладками **«Генератор»** и **«Библиотека страниц»**
2. В библиотеке загружайте скриншоты, давайте им уникальные имена – система опишет страницу через LLM
3. В генераторе настройте поля шаблона, вставьте описание задачи (можно с `{{Имя страницы}}`)
4. Нажмите «Сгенерировать тест-кейсы» и получите структурированный результат

## ⚙️ Быстрый старт

### 1. Клонируйте репозиторий
```bash
git clone https://github.com/Ann-123/Case_Generator.git
cd Case_Generator
```
### 2. Настройте переменные окружения
Создайте файл .env в корне проекта. Пример для Mistral AI:
```
LLM_PROVIDER=mistral
MISTRAL_API_KEY=mistral_ваш_ключ
LLM_MODEL=open-mistral-nemo
VISION_MODEL=pixtral-12b-2409   # модель для распознавания скриншотов
```
### 3. Установите зависимости
```
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
```
### 4. Запустите сервер
```
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```
### 5. Откройте приложение
Перейдите на http://localhost:8000

## 📡 API Endpoints

POST /generate – генерация тест-кейсов
Принимает:
```
{
  "task_text": "Описание задачи... Можно с {{Страница логина}}",
  "fields": ["Название", "Предусловия", "Шаги", "Ожидаемый результат"]
}
```
``` POST /pages/upload``` – загрузка скриншота
Отправьте ```multipart/form-data``` с полями ```file``` (изображение) и ```name``` (уникальное имя).
Модель вернёт описание страницы, которое сохранится в SQLite.

```GET /pages/list``` – список сохранённых страниц
```DELETE /pages/{name}``` – удалить страницу

## 🛡️ Тестирование

Проект покрыт интеграционными и юнит-тестами с использованием `pytest` и `TestClient`

**Всего:** 67 тестов (все проходят успешно).

### Структура директории тестов
```text
tests/
├── __init__.py
├── conftest.py          # Конфигурация окружения (переменные среды MISTRAL_API_KEY и др.)
├── test_database.py     # 22 теста: функции clean_page_name, CRUD, регистронезависимый поиск
├── test_main.py         # 22 теста: функции build_dynamic_model, replace_placeholders, GenerateRequest
└── test_api.py          # 23 теста: проверка API-эндпоинтов (pages + generate) с использованием AsyncMock
```

Что именно покрыто тестами
- backend/database.py: логика очистки имени, добавление/обновление/удаление страниц, получение страницы по имени и списком, регистронезависимость, обработка спецсимволов в поисковом запросе
- backend/main.py:
  - ```build_dynamic_test_case_model``` — валидация всех возможных комбинаций полей
  - ```replace_placeholders``` — замена плейсхолдеров (один, несколько, отсутствующий, с пробелами, пустое описание, юникод)
  - ```GenerateRequest``` — валидация входящих запросов
- API (тестирование через ```TestClient```):
  - Загрузка скриншотов (с моком Vision API)
  - Получение списка страниц, удаление, генерация тест-кейсов (с моком LLM)
  - Валидация и обработка ошибок (пустой запрос, дубликаты, невалидный JSON, сбои самой модели)

## Запуск тестов
Для запуска всех тестов с подробным выводом (verbose) выполните команду:

```
python3 -m pytest tests/ -v
```
