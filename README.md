# 🧪 QA Test Case Generator

**Микро-SaaS приложение (MVP) для автоматической генерации тест-кейсов и чек-листов на основе текстового описания и визуального анализа страниц через LLM (Mistral).  
Включает в себя:

- **Генератор тест-кейсов** с поддержкой:
  - динамических полей (настраиваемый шаблон);
  - подстановки описаний страниц через плейсхолдеры `{{Имя страницы}}`;
  - **встроенного чек-листа** (позитивные / негативные проверки) – можно выбрать нужные пункты и сгенерировать кейсы только по ним.
- **Библиотека страниц** – загрузка скриншотов UI, их описание через Vision LLM (Pixtral) и сохранение в БД для дальнейшего использования.

---

## ✨ Возможности

- **Конструктор шаблона** – добавляйте/удаляйте любые поля тест-кейса (название, предусловия, шаги, ожидаемый результат, окружение и т.д.)
- **Генерация через LLM** – поддерживается несколько провайдеров (OpenAI, Mistral, Groq, OpenRouter, Ollama)
- **Библиотека страниц** – загрузка скриншотов, автоматическое описание страницы через мультимодальную модель (Mistral Pixtral / LLaVA) и сохранение в SQLite
- **Плейсхолдеры в описании задачи** – пишите `{{Имя страницы}}` в поле задачи, и система подставит сохранённое описание
- **Строгая валидация ответа** – динамическая Pydantic-схема гарантирует, что модель вернёт данные только с указанными атрибутами
- **Копирование в буфер** – результат отображается в структурированном виде и может быть скопирован как JSON
- **Лёгкий фронтенд** – один HTML-файл с Tailwind CSS и ванильным JS (без фреймворков)
- **Готовность к деплою** – FastAPI отдаёт и бэкенд, и статику
- **Интерактивный чек-лист проверок** - автоматическая генерация структурированного списка проверок (позитивные/негативные & функциональные/нефункциональные) с возможностью выбора нужных пунктов перед генерацией тест-кейсов
- **Выборочная генерация** - каждый пункт чек-листа превращается в 1+ детализированных тест-кейсов с автоматическим проставлением типа и категории проверки

## 🖥️ Использование

**Вкладка «Генератор» (единый мастер-процесс)**

1. Настройте поля тест-кейса – по умолчанию: «Название», «Предусловия», «Шаги», «Ожидаемый результат». Можно добавлять свои поля (кнопка «+ Добавить поле»).

2. Введите описание задачи – можно использовать плейсхолдеры вида {{Имя страницы}}.
Список доступных страниц отображается под полем ввода – клик по кнопке «Вставить» подставит плейсхолдер в текст.

3. Сформируйте чек-лист (опционально) – нажмите кнопку «Сформировать чек-лист».
Появится блок с пунктами проверок, сгруппированными по областям и разделёнными на позитивные и негативные.

4. Выберите нужные пункты – можно выделить отдельные чекбоксы, целые области или колонки (галочка над колонкой).
Если ничего не выбрать – генерация тест-кейсов пройдёт без привязки к чек-листу.

5. Нажмите «Сгенерировать тест-кейсы» – результат появится в блоке «Результат».
Если были выбраны пункты чек-листа, в запрос автоматически добавится поле «Тип» (Позитивный/Негативный), и кейсы будут сгенерированы только по выбранным проверкам.

6. Скопируйте результат – кнопка «Скопировать всё в буфер» сохраняет JSON с тест-кейсами.

**Вкладка «Библиотека страниц»**

1. Загрузка новой страницы:
   - Выберите изображение (скриншот) и введите уникальное имя.
   - Нажмите «Загрузить и распознать» – изображение отправится в Vision LLM, описание страницы сохранится в БД, а само изображение будет сжато и сохранено в backend/static/uploads/.

2. Список сохранённых страниц – отображается ниже. Каждую страницу можно удалить (крестик справа).

## 🛠 Технологии

- **Backend**: Python 3.10+, FastAPI, Pydantic, aiosqlite (асинхронный SQLite)
- **Frontend**: HTML5, Tailwind CSS (CDN), Vanilla JS
- **LLM**: Mistral API (текстовые модели + Pixtral для зрения)
- **База данных**: SQLite (с асинхронным драйвером)
- **Оптимизации**:
  - батчинг запросов к БД для плейсхолдеров (устранение N+1);
  - кэширование описаний страниц в памяти;
  - сжатие изображений перед отправкой в Vision API;
  - асинхронные вызовы БД (aiosqlite);
  - унификация парсинга JSON от LLM.

## ⚙️ Быстрый старт

### 1. Клонируйте репозиторий
```bash
git clone https://github.com/Ann-123/Case_Generator.git
cd Case_Generator
```
### 2. Настройте переменные окружения
Создайте файл .env в корне проекта. Пример для Mistral AI:
```
LLM_PROVIDER=mistral
MISTRAL_API_KEY=mistral_ваш_ключ
LLM_MODEL=open-mistral-nemo
VISION_MODEL=pixtral-12b-2409   # модель для распознавания скриншотов
```
### 3. Установите зависимости
```
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
```
### 4. Запустите сервер
```
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```
### 5. Откройте приложение
Перейдите на http://localhost:8000

## 📡 API Endpoints

- ```POST /generate``` – генерация тест-кейсов.
Тело: ```{ "task_text": "...", "fields": ["Название", ...], "checklist_items": [...] }``` (последнее – опционально).

- ```POST /generate-checklist``` – генерация чек-листа.
Тело: ```{ "task_text": "..." }```.

- ```POST /pages/upload``` – загрузка страницы (multipart/form-data: file, name).

- ```GET /pages/list``` – список всех страниц.

- ```DELETE /pages/{name}``` – удаление страницы.

Подробные схемы запросов и ответов можно посмотреть в коде backend/main.py.
